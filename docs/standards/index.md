## 🔧 規範建立流程

### 新增規範時的標準流程
1. **確認需求**：評估是否需要新的規範
2. **選擇位置**：
   - 可執行指令 → `.claude/commands/`
   - 專案規範 → `docs/standards/`
   - 技術架構 → `docs/architecture/`
3. **建立文件**：遵循現有格式與風格
4. **更新索引**：在本檔案中新增規範項目
5. **更新 CLAUDE.md**：確保 Claude Code 可讀取

### 規範維護原則
- **版本控制**：所有規範變更需要版本控制
- **影響評估**：變更前評估對現有代碼的影響
- **團隊同步**：規範變更需要通知所有團隊成員
- **定期檢視**：定期檢視規範的實用性與時效性
